## Commands

### Root (Turborepo)
```bash
npm run dev                        # Start all apps in parallel
npm run dev -- --filter=pip        # Start a single app (pip, web, necrobloom)
npm run build                      # Build all apps
npm run lint                       # Lint all packages
npm run format                     # Format TS/TSX/MD with Prettier
npm run storybook                  # Launch Storybook for @finnminn/ui
npm run bootstrap                  # One-time local env setup (Docker, SSL, settings)
npm run emulators:start            # Start Cosmos DB + Azurite Docker containers
npm run emulators:stop             # Stop emulators
```

### Backend (Kotlin/Gradle — run from `apps/pip/api` or `apps/necrobloom/api`)
```bash
./gradlew azureFunctionsRun        # Start local Azure Functions (port 7071)
./gradlew build                    # Build Kotlin project
```

### Testing (Pip has Vitest)
```bash
npm run test --filter=pip          # Run all tests
npm run test -- --filter=pip -- --run src/path/to/file.test.tsx  # Single test file
```

## Architecture

**Turborepo monorepo** with npm workspaces. Each app in `apps/` is an independent SPA deployed to Azure Static Web Apps; backends are Kotlin Azure Functions deployed separately.

### Apps (`apps/`)
- `web` — Landing page (`finnminn.com`), port 5175
- `pip` — Habit tracker (`pip.finnminn.com`), port 5173, has Vitest tests
- `necrobloom` — Plant tracker (`necrobloom.finnminn.com`), port 5174
- `eventhorizon` — Educational static site (VitePress + MathJax)

### Shared Packages (`packages/`)
- `@finnminn/ui` — React component library implementing the **PixelGrim** design system
- `@finnminn/auth` — MSAL wrapper: `AuthProvider`, `useAuth` hook (Microsoft Entra ID)
- `@finnminn/config` — Shared Tailwind, ESLint, TypeScript configs

### API Routing
- **Local**: Vite proxy → `localhost:7071`; mock `x-ms-client-principal` header injected automatically
- **Production**: Frontend calls absolute `VITE_API_URL` → APIM Gateway → Azure Function Apps (required because SWA Free tier doesn't support "Bring Your Own API")

## Design System: PixelGrim

**Aesthetic:** Whimsical Gothic Tech — 8-bit CRT nostalgia + gothic magic + terminal utility.

**Mandatory rules for all UI work:**
- All text **must** use `<Typography />` sub-components (H1, H2, Body, Code, etc.)
- No hardcoded colors — use Tailwind tokens: `text-witchcraft`, `text-ectoplasm`, `text-vampire`, `text-toxic`, `bg-void`, `bg-surface`
- Borders: always `border-2`; radius: always `rounded-none`; shadows: `shadow-pixel` or `glow-[color]`
- `glow={true}` must **never** be used on text `size="xs"` or `"sm"`
- Minimum `12px` font size for retro fonts (VT323, Press Start 2P) — use `size="xs"` prop
- All apps must use the shared `CommandBar` within a `Layout` wrapper, with a link back to `/apps`

### New Component Checklist (`packages/ui`)
1. Define the component in `packages/ui/styleguide.toml` first
2. Implement in `src/components/`
3. Create `src/stories/ComponentName.stories.tsx` (follow Storybook hierarchy: Primitives/, Containers/, Navigation/, Media/, System/)
4. Add to `src/stories/DesignSystem.stories.tsx` (AllComponents "Kitchen Sink")
5. Export from `src/index.tsx`

## Local Development

`local.settings.json` files (gitignored) are auto-generated by `npm run bootstrap` from `*.example.json` templates.

**Authentication:** If the app shows a login page or `NO_ACTIVE_ACCOUNT` errors, stop and ask the user to sign in manually. Automated workflows cannot proceed without an active session.

**Ports:**
- Frontend: 5173 (pip), 5174 (necrobloom), 5175 (web)
- Backend: 7071 (Azure Functions)
- Storybook: 6006
